# Internationalization (i18n) Support

## Setup Guide

The dashboard uses `react-i18next` for internationalization support with four languages:
- English (en) - base
- Spanish (es)
- Portuguese (pt)
- Mandarin Chinese (zh)

### Installation

```bash
npm install react-i18next i18next
```

### Configuration

The i18n system is initialized in `src/i18n/config.ts`:

```typescript
import { initI18n, changeLanguage } from "@/i18n/config";

// Initialize on app startup
initI18n();
```

## How to Add a New Language

1. Create a new translation file in `src/locales/{lang-code}.json`
2. Add the language to `SUPPORTED_LOCALES` in `src/i18n/config.ts`
3. Add the language option to `LanguageSwitcher` component
4. Import and register in the i18n resources

## Translation Key Conventions

Keys follow a dot-notation hierarchy:

```
navigation.dashboard      # Navigation labels
buttons.save             # Button text
forms.placeholder.search   # Form placeholders
labels.latency           # Labels and statuses
errors.generic          # Error messages
fee.estimatedFee        # Fee-related strings
```

### Key Categories

- `navigation` - Menu and navigation labels
- `buttons` - Action button text
- `status` - Status indicators (operational, degraded, outage)
- `forms.placeholder` - Form input placeholders
- `labels` - UI labels and descriptors
- `errors` - Error messages
- `fee` - Fee estimation related strings
- `theme` - Theme selector options
- `table` - Table headers and empty states
- `time` - Time range selectors

## Locale Formatting Rules

### Number Formatting

```typescript
new Intl.NumberFormat(locale).format(1234.5)
```

### Date Formatting

```typescript
new Intl.DateTimeFormat(locale).format(date)
```

### Currency Formatting

```typescript
new Intl.NumberFormat(locale, { style: "currency", currency: "USD" }).format(amount)
```

### RTL/LTR Support

Mandarin Chinese (zh) is left-to-right. For future RTL languages, wrap content with:

```typescript
const isRTL = ["ar", "he"].includes(currentLocale);
```

## Language Switcher Component

Located at `src/i18n/LanguageSwitcher.tsx`:

```tsx
import { LanguageSwitcher } from "@/i18n/LanguageSwitcher";

<LanguageSwitcher />
```

Features:
- Dropdown with flag icons and language names
- Persists selection to localStorage (`fluid-admin-language`)
- Fully keyboard accessible with ARIA labels
- Closes on outside click

## Usage in Components

```typescript
import { useTranslation } from "react-i18next";

function MyComponent() {
  const { t } = useTranslation();
  return <h1>{t("navigation.dashboard")}</h1>;
}
```

## Fallback Behavior

- Missing keys fall back to English
- Unknown languages default to English
- Use `t(key, fallback)` for custom fallback text
